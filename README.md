# TheoWebsite

Personal website ready for GitHub Pages.

## Edit your details

Before publishing, update these values in `index.html`:
- `Theo` (name in logo and footer)
- About section text
- Project titles, descriptions, and links
- Email address (`you@example.com`)

## Run locally

You can open `index.html` directly in your browser, or use VS Code Live Server.

## Publish with GitHub Pages (GitHub Actions)

1. Create a new GitHub repository and push this folder to the `main` branch.
2. In GitHub, open your repository settings.
3. Go to **Pages**.
4. Under **Build and deployment**, set **Source** to **GitHub Actions**.
5. Push to `main` (or rerun the workflow in the **Actions** tab).

After deployment, your site will be available on your repository's Pages URL.

## If you want a custom domain

1. Add your domain in GitHub Pages settings.
2. Create a `CNAME` file in the project root with your domain name.
3. Update DNS records to point to GitHub Pages.
